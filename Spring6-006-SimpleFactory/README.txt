简单工厂模式 SimpleFactory Pattern

1. 简单工厂模式是一种特殊的工厂方法模式，又被称为 静态 工厂方法模式

2. 简单工厂模式

    1. 抽象产品角色

    2. 具体产品角色

    3. 工厂类角色

3. 简单工厂模式解决的问题

    1. 客户端不需要关心对象的创建细节，需要使用对象的时候，向工厂索要即可，初步实现类职责分离

4. 简单工厂模式的缺点

    1. 如果有了新的产品需要拓展，则需要修改原来的代码，这违反了开闭原则

    2. 工厂类责任重大，因为它负责所有产品的生产，不能出问题。如果工厂类出了问题，整个系统必然瘫痪。
       (不能把所有鸡蛋放一个篮子类)
